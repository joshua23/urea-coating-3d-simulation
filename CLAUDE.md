# CLAUDE.md

此文件为 Claude Code (claude.ai/code) 在此代码库中工作时提供指导。

## 项目概述

这是一个**尿素包膜生产线3D全景模拟系统** - 一个纯前端Web应用程序，用于可视化和模拟工业设备运行。

## 技术栈

- **纯前端**: 无需后端服务器
- **核心技术**: 原生 HTML5、CSS3、JavaScript (ES6+)
- **3D库**: Three.js (v0.155.0) 通过CDN加载
- **无构建流程**: 直接在浏览器中执行，无需编译

## 开发命令

由于这是一个静态HTML/CSS/JS项目，没有构建系统：

```bash
# 打开项目 - 直接在浏览器中打开 index.html
open index.html

# 开发时使用实时重载（可选，使用任何静态服务器）
python3 -m http.server 8000  # Python
# 或
npx serve                     # Node.js serve包
```

## 架构概述

### 文件结构
- **index.html**: 主应用入口，包含所有HTML结构和设备布局
- **styles.css**: 所有样式、动画和3D转换
- **script.js**: 核心应用逻辑和设备控制系统

### 核心组件

1. **设备系统**: 9个主要生产设备单元，每个都具有：
   - 独特的3D CSS模型表现
   - 运行/停止/故障状态的视觉反馈
   - 参数控制（速度、旋转、振动等）
   - 详细的技术规格说明

2. **控制系统**：
   - 手动控制（单个设备启动/停止）
   - 自动模式（设备状态顺序变化）
   - 通过滑块实时调整参数
   - 每个设备5种故障类型的故障模拟

3. **UI面板**：
   - 左侧面板：设备信息显示和故障模拟
   - 中央面板：3D生产线可视化
   - 右侧面板：实时监控和参数控制
   - 顶部控制：全局操作按钮

### 关键JavaScript对象

- `equipmentInfo`: 每个设备的技术规格
- `equipmentParams`: 当前参数值和状态
- `faultStates`: 活跃故障状态跟踪
- 设备交互、参数变更和故障模拟的事件处理器

### 动画系统

基于CSS的动画控制通过：
- `.running` 类用于运行状态
- `.fault-*` 类用于不同的故障类型
- 通过CSS变量动态调整动画速度
- 3D变换实现透视和深度效果

## 重要实现细节

- 设备ID在HTML、CSS和JS中保持一致（例如：`material-feeding`、`z-elevator`）
- 动画速度通过动态修改CSS animation-duration控制
- 故障状态应用视觉滤镜和特殊动画
- `.next`文件夹是Next.js构建产物，但项目作为静态HTML运行
